2005/04/19

2005/05/19

SMS encounter status codes (encStsCd) are: A (active); D (discharged); C (completed); X (deleted). These codes map to the Oacis status codes (encOacisStsCd) t (active), f (discharged, completed) and x (deleted) respectively.

2005/04/19

2005/05/19

Note that there is an additional Oacis status of c (cancelled) which does not map directly to an equivalent SMS status. The SMS status can remain as A (active) or, in the case of a cancelled admit, be P (pre-admit).

2005/04/19

2005/05/19

The Oacis status (encOacisStsCd) is derived from a combination of the SMS status and other values (such as 2APST for regular outpatients or C2549 [=0] for General AmbCare cases). So it is possible in the DW to have encStsCd=A and encOacisStsCd=c or x.

2005/04/19

2005/05/19

Note that AmbCare no-shows are being mapped to a encOacisStsCd of x (deleted) instead of c (cancelled), resulting in a scenario where encStsCd = A while encOacisStsCd = x.

2005/06/08

2005/09/26

Do not use encounters where both encOacisStsCd and encStsCd are not valued. This is data entry problem in the source systems. But the services that belong to those encounters can be used.

2005/07/27

2005/09/26

Another column called encStsCd also exists and should be used whenever it is valued. When encStsCd is null though, encOacisStsCd can be used.
