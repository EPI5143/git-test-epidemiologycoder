Database Naming Conventions and Standards

Database Table Names
A series of meaningful word(s) to describe the use of the table should be used as table name. The first letter of each table name will represent the model (N indicates the Normalized model). The next word (which immediately follows the first letter) should not be capitalized, but each subsequent word should be capitalized. All words should be concatenated with no joining character. (e.g. NserviceReportText)

Database Column Names
A two, three or four character prefix representing the table in which the element resides must be used for each column name, without capital letters. Each subsequent word following the prefix must be capitalized. (e.g. encCampusCd; svrtLineNum)

Design Standards
Tables will have identity fields assigned with the table prefix (as described for column names) and the name WID which stands for Warehouse Identifier (e.g. facWID in the Nfacility table). These will be defined as integer and will generally be the primary key for the table. These will be defined as integer and will generally be the primary key for the table. Foreign keys will be prefixed for the table in which they reside and carry the name of the field from the parent table. (e.g. sacFacWID in the NsourceApplicationCode table is the foreign key for the facWID primary key).

Standard Table Columns
All tables will also have columns for create date and time (WCT), update date and time (WUT) and update user (WUU). These indicate when the entries were created (WCT=Warehouse Create Time) and when the entries were last updated (WUT=Warehouse Update Time) and who or what program last updated the entry (WUT=Warehouse Update User). These will follow the naming convention described above (e.g. facWCT).

It is important to understand that these columns reflect create / update activity in the Data Warehouse and not in the source system. Also, sometimes the Data Warehouse update process is such that a new row is always re-created rather than being updated.
